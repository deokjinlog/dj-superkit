01개요

dj-superkit 플러그인을 쓰다 보면 "업데이트했는데 슬래시가 안 떠", "이거 dj-superkit 밖에 있는 거 아냐?", "/list-skills 가 왜 비어 있어?" 같은 혼란이 자주 생긴다. 원인은 대부분 명령(command)과 스킬(skill)의 차이, 플러그인 로딩 시점(재시작), 스킬 빌더의 스코프/출처표식 모델 세 가지를 헷갈려서다. 이 노트는 그 세 축을 한 번에 정리한다.

02핵심 개념

1명령(command) vs 스킬(skill)

명령 (command)스킬 (skill)
파일 위치 commands/<이름>.md skills/<슬러그>/SKILL.md
호출 방식 슬래시로 명시 호출 (/learn) 자연어로 자동 발동 ("이거 학습 정리해줘")
발동 근거 사용자가 직접 / 침 description 안의 트리거 문구 매칭
슬래시 호출 됨 됨 (/dj-superkit:learning-notes)
  • 핵심: 한 기능을 명령·스킬 둘 다로 노출할 수 있다.
  • 예: /learn 은 얇은 명령 파일이 learning-notes 스킬을 명시 호출하도록 짜여 있다. 그래서 /learn 슬래시로도 부르고, "이거 학습 정리해줘" 라고 말해도 자동으로 뜬다.

2플러그인 로딩 = 세션 시작 시점 스냅샷 (재시작 필요)

  • claude plugin update 를 하면 새 버전이 캐시에 설치된다:
    ~/.claude/plugins/cache/dj-superkit/dj-superkit/<버전>/
  • 하지만 이미 실행 중인 세션은 세션이 시작된 시점의 플러그인 상태를 그대로 들고 간다.
  • 그래서 업데이트 직후 항상 "Restart to apply changes" 가 뜬다.
  • 새 명령/스킬을 쓰려면 새 세션을 켜야 로드된다. 현재 세션에서 아무리 슬래시를 쳐도 안 뜬다.

3슬래시 접두사 dj-superkit: 는 선택 사항

  • 플러그인 명령의 정식 이름은 /dj-superkit:<이름> 이다.
  • 하지만 이름이 다른 것과 안 겹치면 Claude Code 가 짧은 형태 /<이름> 으로도 보여준다.
  • 즉 /learn 과 /dj-superkit:learn 은 같은 명령이다.
  • 짧게 보인다고 "dj-superkit 밖에 있는" 게 아니다. 접두사는 소속 표시가 아니라 이름 충돌을 막는 구분자일 뿐.

4스킬 빌더 3종과 스코프 모델

/new-skill 로 만든 커스텀 스킬은 두 스코프 중 하나에 저장된다:

스코프저장 위치
프로젝트<현재 프로젝트>/.claude/skills/<슬러그>/
전체(글로벌)~/.claude/skills/<슬러그>/
  • 만들 때 출처 표식 파일 .dj-superkit-skill.json 이 함께 생긴다.
  • /list-skills: 출처 표식이 있는 스킬만 프로젝트+글로벌 두 스코프에서 조회. 읽기 전용. 다른 프로젝트 폴더는 스캔 안 함.
  • /remove-skill: 출처 표식이 없으면 무조건 차단(--force 로도 우회 불가) → 남의 스킬을 실수로 못 지운다.
    • 기본은 .removed-<시각> 으로 이름만 바꿔 치워둠(복구 가능)
    • --force = 완전 삭제(복구 불가)
    • --dry-run = 뭐가 지워질지 미리보기

주의: 내장 스킬(brainstorm, learning-notes 등)에는 출처 표식이 없다. 스킬 빌더가 만든 게 아니라 플러그인에 원래 들어 있는 거라서. 그래서 /list-skills 에 안 뜨는 게 정상이다.

5러닝노트 저장 위치

  • 학습노트는 현재 작업 폴더(cwd) 기준 docs/learning/<날짜>-<슬러그>/ 에 저장된다.
  • dj-superkit 레포 안에서 작업 중이면 → dj-superkit 안에 저장된다.
  • 다른 프로젝트에서 작업 중이면 → 그 프로젝트 안에 저장된다 (중앙 폴더 없음).

03헷갈렸던 점 Q&A

Q. 플러그인 업데이트했는데 왜 슬래시 메뉴에 새 명령이 안 떠?

A. 현재 세션이 세션 시작 시점의 옛 플러그인 상태를 들고 있어서. 업데이트는 캐시에만 반영됐고, 세션엔 아직 로드 안 됨. 업데이트할 때마다 뜨는 "Restart to apply changes" 가 그 신호다. 새 세션을 켜면 뜬다.

Q. /schedule 같은 건 뜨는데 왜 dj-superkit 명령만 안 떠?

A. /schedule 등은 Claude Code 내장 명령이라 플러그인 로딩과 무관하게 항상 있다. dj-superkit 명령은 플러그인 소속이라 세션 재시작 전엔 안 뜬다.

Q. /learn 이 /dj-superkit:learn 으로 안 감싸져 보여. 밖에 있는 거 아냐?

A. 아니다. 이름이 안 겹치면 짧은 /learn 으로 보여줄 뿐, 같은 dj-superkit 명령이다. /dj-superkit:learn 풀네임으로 쳐도 똑같이 동작한다.

Q. /list-skills 를 쳤는데 왜 아무것도 안 나와?

A. /list-skills 는 내가 /new-skill 로 직접 만든 커스텀 스킬만 본다(출처 표식 있는 것만). 그렇게 만든 게 0개면 비어 있는 게 정상. dj-superkit 내장 28개 스킬은 대상이 아니다 — 표식이 없기 때문.

Q. 명령이랑 스킬은 뭐가 달라? 왜 둘 다 만들어?

A. 명령은 슬래시로 내가 직접 부르는 것, 스킬은 자연어에서 자동으로 떠오르는 것. 같은 기능을 둘 다로 노출하면, 슬래시를 기억 못 해도 말로 부를 수 있고(자동), 정확히 부르고 싶을 땐 슬래시로 부른다(명시). /learn + learning-notes 조합이 그 예.

Q. 학습노트는 어디에 저장돼?

A. 지금 작업 중인 폴더 안 docs/learning/ 에. dj-superkit 레포에서 작업 중이면 dj-superkit 안에 저장된다.

04도식

명령 vs 스킬 호출 흐름

diagram
[ /learn 슬래시 입력 ] ──명시 호출──┐
                                    ├──▶  learning-notes 스킬 실행
[ "이거 학습 정리해줘" ] ─자동 발동─┘
        (description 트리거 매칭)

플러그인 업데이트 → 반영까지

diagram
claude plugin update
        │
        ▼
캐시에 새 버전 설치  (~/.claude/plugins/cache/.../0.6.0/)
        │
        ▼
"Restart to apply changes"   ←  현재 세션은 아직 옛 상태
        │
        ▼
새 세션 켜기  ──▶  새 명령/스킬 로드 완료

05한 줄 요약

1

업데이트 후 슬래시 안 뜸 = 재시작 필요 (세션은 시작 시점 스냅샷).

2

/learn = /dj-superkit:learn (짧게 보여도 dj-superkit 소속).

3

명령은 슬래시로 부르고, 스킬은 말로 떠오른다 — 한 기능을 둘 다로 낼 수 있음.

4

/list-skills 는 /new-skill 로 만든 것만 본다 (내장 스킬은 표식 없어서 안 뜸).

5

러닝노트는 현재 작업 폴더 안 docs/learning/ 에 저장.
