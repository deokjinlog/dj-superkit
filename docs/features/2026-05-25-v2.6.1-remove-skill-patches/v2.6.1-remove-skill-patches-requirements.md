## §1. 배경

v2.6.0 push + reload 직후 dogfood 검증에서 다음 두 종류 issue catch.

**A. 짝 명령 부재** — `/new-skill` 빌더는 생겼지만 만든 skill 을 지우는 짝 명령 (`/remove-skill`) 이 없음. 사용자가 글로벌 `~/.claude/skills/<slug>/` 누적 후 정리하려면 수동 `rm -rf` 입력 필요. 빌더 UX 비대칭.

**B. v2.6.0 자체의 critical 후보 3건** — release 직후 메인 검토에서 catch.

1. **command 가 skill list 에 노출됨** — reload 후 35 → 36 skills 카운트 증가. `dj-superkit:new-skill` description 이 "skill 자동 생성" 류라 평상 대화에서 자동 발동 위험. META-BUILDER-HANDOFF.md §5 "command 로 만들면 자동 발동 X" 가정 깨짐.
2. **step 폭주 alert 의 `--force` 안내 오류** — `commands/new-skill.md` § 3-2 본문 "그대로 진행하시려면 `--force` 옵션 추가" — `--force` 는 덮어쓰기용. 사용자 혼동 유발 명시 오류.
3. **`--force` 백업 instruction 모호** — § 5-3 본문 "Write 도구로 copy 하거나 별도 안내". Claude Write 도구는 새 파일 쓰기라 copy 안 됨. Read 후 다른 경로 Write 가 정확. 안전성 핵심 (D-T5) 손상 위험.

→ 4 항목 (A + B 3건) 통합 patch v2.6.1 — atomic. v2.6.0 release 직후라 dogfood 시작 전 회수 권장.

### D3. `/remove-skill` 의 default 동작 — safe-rename (백업 후 hard-delete X)

- 기본 동작: `~/.claude/skills/<slug>/` 를 `~/.claude/skills/<slug>.removed-<YYYYMMDDHHMMSS>/` 로 rename

- 회복: 사용자가 `mv <slug>.removed-... <slug>` 로 복원 가능

### D4. `/remove-skill` 의 `--force` 옵션 — 백업 없이 hard-delete

- 명시 시 `rm -rf ~/.claude/skills/<slug>/` 효과 (실제는 Bash 도구로 rm 호출)
- 사용자가 회복 의도 없을 때만 사용 권장
- abort 메시지에 `--force` 명시 옵트인 안내

## §3. 우려 / 해결

### C1. `--force` hard-delete 가 사용자 실수로 데이터 손실?

- **해결**: D4 — 옵트인 플래그 명시. abort 메시지에 회복 불가 명시. default 는 safe-rename (D3) 으로 회복 가능.

### C2. safe-rename 후 자동 발동 차단 보장?

- **해결**: 디렉토리 rename 만으로는 Claude Code 가 skill 자동 catch 안 함을 가정하지만 명확 X. 사용자가 rename 후 `/reload-plugins` 한 번 권장 — 빌더 본문에 보고 메시지로 안내. v2.6.x 후속 dogfood 에서 catch 영역.

### C3. `/new-skill` 자동 발동 위험 — D6 patch 만으로 충분?

- **해결**: description 좁히기 + "명시 호출 only" 톤 우선. 100% 보장 X — Claude 의 자동 발동 판단 LLM. v2.6.x 후속에서 자동 발동 dogfood 결과 catch 후 추가 patch 여지.

### C4. step 폭주 alert 정정 후 다시 사용자 혼동?

- **해결**: D7 의 새 메시지가 "다시 호출 / 두 번 호출" 2 옵션 명확. `--force` 무관 명시. 빌더 본문 8 섹션 안 다른 곳도 같이 catch 했는지 추가 grep.

### C5. v2.6.0 patch 와 remove-skill 신규 동시 release 가 너무 무겁다?

- **해결**: 본 patch 4 항목이지만 모두 commands/ 영역 + 결합 메모만. 다른 skill body / scripts / hooks 영향 0. 한 묶음 release 가 자연.

### C6. dogfood 시작 안 한 상태로 후속 patch — release 신뢰성?

- **해결**: v2.6.0 release 직후 catch + 회수 흐름. 명확한 회귀 / 안전성 catch 라 dogfood 전 회수가 정직. v2.6.1 도 dogfood 후 catch 시 v2.6.2 재 patch 여지.
