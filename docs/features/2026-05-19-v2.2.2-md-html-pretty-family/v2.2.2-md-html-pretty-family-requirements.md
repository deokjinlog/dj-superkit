# v2.2.2 — `.md` / `.html` pretty 명령 family 완성 + 스킬 리네임

Mode: Socratic (auto-flow)

## 배경

v2.2.1 에서 `docs-pretty` 가 `.md` format-only pass 책임을 잃고 `.html` 만 생성하게 됨. 그 결과 다음 4 가지 정합 누수 발생:

1. **`.md` 자체 정형 수단 부재** — 사용자가 가끔 `.md` 를 외부 공유 (PR 본문 paste / 포트폴리오 / 인쇄 등) 하고 싶을 때 손으로 다듬어야 함.
2. **`docs-pretty` 이름이 책임을 대변 안 함** — `.html` 만 생성하는데 "docs pretty" 표현은 거짓말. 신규 사용자가 `.md` 가공이라 오해.
3. **`/regen-html` 이름의 "처음부터 다시" 뉘앙스** — 매번 디자인이 달라질 거라는 오해. 실제 정상 사용은 디자인 보존 + 내용 sync.
4. **change-propagation 끝 `.html` stale 인지 불가** — 손으로 호출 잊어버림 (자동 호출은 v2.2.1 단순화 무화 위험).

## 핵심 결정

### 채택 approach — A1: 5 항목 atomic patch (백로그 그대로)

| 항목 | 산출물 | 모드 |
|---|---|---|
| 1. `/pretty-md` 신규 | `commands/pretty-md.md` | fire-and-forget subagent (Sonnet format-only `.md`) |
| 2. `/sync-html` 신규 (+ `--rebuild`) | `commands/sync-html.md` | fire-and-forget subagent (Sonnet) |
| 3. `docs-pretty` → `generating-html` 리네임 | `skills/docs-pretty/` → `skills/generating-html/` | 폴더 + frontmatter + 자기지칭 일괄 교체 |
| 4. change-propagation 끝 힌트 | `skills/change-propagation/SKILL.md` 마지막 한 줄 | 자동 호출 X — 안내문만 |
| 5. 단어 일괄 교체 | grep 0 (og-* + 백로그 history 제외) | `docs-pretty` / `regen-html` 참조 모두 |

**왜 A1 (분할 X)**: 5 항목 한 곳만 누락 시 회귀 (단어 grep 일관성 깨짐, 신규 명령은 있는데 구 명령 잔존 → 사용자 혼란). 백로그 작성자가 atomic patch 룰 명시.

**대안 A2 (리네임 먼저, 신규 명령 별도 release)** — 거부: 리네임 + 단어 교체는 release 분할 의미 0 (사용자 가시 변화 X 까지 다중 release 부담). 신규 명령도 같은 흐름 (`.md` / `.html` pretty 책임 정합) 이라 분할 시 정합 누수 카운트만 ↑.

**대안 A3 (`--rebuild` 제거)** — 거부: 디자인 변경 필요 케이스 (e.g., 디자인 실험 / 사용자 디자인 불만) 가 dogfood 검증 안 됨. 일단 포함 + 안 쓰이면 v2.2.3 제거. 백로그 명시 답습.

### 미정 4건 자동 결정

| 항목 | 결정 | 이유 |
|---|---|---|
| `/pretty-md` 디바운스 | **디바운스 X** (매번 dispatch) | opt-in 명시 호출만 — 자동 발동 경로 없으므로 디바운스 의미 X. `/regen-html` (자동 호출 가능) 의 3초 디바운스 패턴과 다름. |
| `/sync-html --rebuild` 유용성 | **포함** (dogfood 후 제거 옵션) | 디자인 실험 / 사용자 디자인 불만 케이스 cover. v2.2.3 dogfood 후 제거 가능. |
| `/pretty-md` 결과 게이트 | **그냥 덮어쓰기** (사용자 git diff 확인) | fire-and-forget 기조 답습. 게이트 추가 시 메인 흐름 차단 → v2.2.1 단순화 무화. 백로그 안 답습. |
| deprecated 안내 | **짧은 1회 안내 + fallback 없음** | 구 `docs-pretty` / `/regen-html` 호출 시 한 줄 "이 명령은 v2.2.2 에서 X 로 교체됨" 안내 + 자동 변환 X. 사용자가 신규 명령으로 직접 갱신. fallback shim 추가 시 deprecated 영구화 위험. |

### 책임 분리 — `.md` vs `.html`

| 책임 | 명령 | 발동 |
|---|---|---|
| `.md` source-of-truth 작성 | spec workflow (brainstorming → designing-direction → writing-plans) | 메인 흐름 자동 |
| `.md` 인간 가독성 정형 | `/pretty-md` | 명시 호출만 (자동 X) |
| `.html` 시각화 사이드카 | `generating-html` skill / `/sync-html` | 자동 (첫 spec 작성 시) + 명시 (`/sync-html`) |
| `.html` ↔ `.md` 양방향 sync | **절대 X** | `.md` 가 source |

## 우려 / 해결

| 우려 | 해결 |
|---|---|
| **C1** 리네임으로 인한 회귀 — 기존 사용자가 `docs-pretty` / `/regen-html` 호출 시 깨짐 | deprecated 안내 메시지 1회 (구 명령 trigger 만 catch, 본문 실행 X). 사용자가 신규 명령으로 갱신. |
| **C2** `/sync-html --rebuild` 가 dogfood 검증 안 됨 → 사용자가 의도와 다르게 디자인 변경 | 백로그 명시 답습 + v2.2.3 후보로 제거 옵션. 사용 빈도 dogfood 후 결정. |
| **C3** `/pretty-md` 게이트 없으면 사용자가 정형 결과 마음에 안 들 때 revert 부담 | git diff + git checkout 으로 revert 가능 (백로그 안 답습). 게이트 추가 시 메인 차단 → 단순화 무화. |
| **C4** change-propagation 끝 힌트만 — 사용자가 안내 무시 시 `.html` 영구 stale | 의도된 동작. stale 허용 철학 (사용자 판단). 자동 호출은 v2.2.1 단순화 무화 + change-propagation Acceptance 5번 룰 (자동 호출) 와 충돌. 본 항목은 boundary 조정. |
| **C5** 단어 일괄 교체 grep 의 og-* / 백로그 history false-positive | grep `--exclude-dir` 옵션 / 검증 단계에서 명시 제외. fixture 에 grep 결과 0 검증 룰 박힘. |
| **C6** `generating-html` 이름이 "사람 전용" 책임 안 드러남 | 백로그 명시 답습 (`generating-html` 채택). 대안 `html-companion` / `html-sidecar` 등은 v2.2.3 후보. |
| **C7** `/pretty-md` 와 `/sync-html` 둘 다 호출 시 의도 (양쪽 갱신) 가 흔할지 | v2.2.2 안 함 — `/pretty-both` 같은 결합 슬래시는 v2.2.3 후보 (백로그 Non-goals 명시). |
