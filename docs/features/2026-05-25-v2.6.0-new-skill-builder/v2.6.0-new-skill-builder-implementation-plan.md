## §2. 위험 코드 지점

### R-1 (LLM 분해 결과가 자유 텍스트 의도 미스)

- **위치**: `commands/new-skill.md` `## 3. LLM 분해 (5 단계)` (Task 2)
- **변경**: 5 단계 분해 룰 본문 명시 + R-7 (slug 의도 미스) catch
- **Mitigation**: D6 `--dry-run` 옵션으로 사전 검토 / D4 명시 slug 인자 우선 / 모호 시 prose 1회 묻고 진행 (메인 자체 판단). 룰 본문에 명시.

### R-2 (글로벌 디렉토리 충돌)

- **위치**: `commands/new-skill.md` `## 4. 사전 검증 룰` + `## 5. Write 분기` (Task 2)
- **변경**: 기존 파일 존재 시 `--force` 없으면 abort + 기존 경로 안내. `--force` 있으면 백업 (`.bak-<timestamp>`) 후 덮어쓰기.
- **Mitigation**: 안전 default (abort) + 회복력 (백업).

### R-3 (description 길이 / 트리거 명확성)

- **위치**: `commands/new-skill.md` `## 4. 사전 검증 룰` (Task 2)
- **변경**: D7 휴리스틱 3건 본문 명시 (길이 max 120자 auto trim + 알림 / "Use when" 패턴 warn / 동사 시작 warn)
- **Mitigation**: 자동 trim (강제) + warn (사용자 자유도 보장).

### R-4 (skill 본문에 비밀값 / 토큰 / 하드코딩 경로)

- **위치**: `commands/new-skill.md` `## 4. 사전 검증 룰` + `## 7. 금지` (Task 2)
- **변경**: 자유 텍스트에 `AKIA` / `sk-` / `xoxb-` / API key 패턴 / `/Users/<name>` / `~/Library` 같은 시그니처 catch 시 abort + 사용자 안내
- **Mitigation**: LLM 자체 판단 — 룰 본문에 catch 패턴 명시. 강제 차단 (사용자 catch 후 재시도).

### R-5 (글로벌 vs 다른 플러그인 / 프로젝트 skill 이름 충돌)

- **위치**: `commands/new-skill.md` `## 6. 보고 양식` (Task 2) + `CLAUDE.md` 결합 메모 (Task 1)
- **변경**: 빌더는 글로벌만 검증 (D-T4). 다른 충돌은 보고 메시지 마지막에 "다른 플러그인 / 프로젝트 동일 이름 skill 있으면 자동 발동 우선순위 모호 — 사용자 직접 catch" 안내.
- **Mitigation**: 빌더 latency 보존 (한 위치만 검증) + 사용자 catch 영역 명시.

### R-6 (LLM 분해 step 수 폭주)

- **위치**: `commands/new-skill.md` `## 3. LLM 분해 (5 단계)` (Task 2)
- **변경**: 본문 instruction step 권장 1~7 룰 명시. step 8+ 시 alert (사용자에게 "step 폭주 — 분리 필요" 안내) 또는 빌더 자체 묶음 추론.
- **Mitigation**: 룰 본문 + alert (강제 차단 X).

### R-7 (slug 자동 생성이 의도 미스)

- **위치**: `commands/new-skill.md` `## 3. LLM 분해 (5 단계)` 3 단계 (Task 2)
- **변경**: 명시 slug 인자 우선 (D4). 자유 텍스트 명사구 catch 가 모호 시 prose 한 줄 묻고 진행 (메인 자체 판단). 무한 clarifying 방지 — 1회만.
- **Mitigation**: 명시 인자로 우회 가능 + dry-run 으로 검토 가능.
