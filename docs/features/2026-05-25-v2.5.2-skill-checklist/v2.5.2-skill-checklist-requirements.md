## 배경

v2.5.1 release 진행 세션 도중 사용자가 catch:

> "잠깐 너왜 방금 태스크 생성안하고 했어? 요즘 버그가 태스크 생성 안하고 하는거 같더라.. 이거 한번 살펴봐줄래?"

원인 파악 결과:

- using-superpowers 의 룰: "Has checklist? yes → Create TodoWrite todo per item, no → Follow skill exactly"
- 즉 skill body 안에 `## Checklist` 섹션이 있어야 메인이 자동으로 TaskCreate 발동
- 현재 33 skill 중 4 만 `## Checklist` 섹션 보유 (brainstorming / designing-direction / writing-plans / og-brainstorming)
- 나머지 29 skill 은 미보유 — auto-* / executing-plans / 워크트리 skill 등 사용자가 진행 상황 보고 싶어할 만한 skill 다수 포함

v2.5.1 본 release 진행 자체가 dogfood — 본 catch 가 발견된 그 세션. 메인이 수동으로 4 task 박았지만 본 세션 한정 효과. 다음 세션 진입 시 같은 문제 재발.

v2.5.2 patch 로 9 skill body 에 `## Checklist` 섹션 신규 추가.

### D2. Checklist 항목 = 각 skill 의 Process 단계 매핑

각 skill 의 `## Process` 섹션의 Step 1~N 을 그대로 Checklist 항목으로 1:1 매핑. 신규 룰 도입 X — 기존 Process 흐름을 사용자 시야에 노출하는 단순 패턴.

예시 (auto-brainstorming):

```markdown
## Checklist

- [ ] Step 1 — Slug 추론 + 폴더 생성
- [ ] Step 2 — Socratic clarifying questions (1~5개 적응)
- [ ] Step 3 — AI 자동 approach 선택
- [ ] Step 4 — 산출물 자동 작성 (<slug>-requirements.md)
- [ ] Step 4.5 — generating-html fire-and-forget dispatch (+ 5초 race delay)
- [ ] Step 5 — change-history 자동 (첫 [요구사항-수정] entry)
```

## 우려 / 해결

### R1. 9 skill 의 Checklist 항목이 너무 다양해서 일관성 깨질 위험

D2 가 Process 단계 1:1 매핑이라 각 skill 의 Step 수가 다 다름 (5~10 항목). 사용자 시야에 일관성 없는 UX.

**해결**: 각 skill 의 Step 헤더 + 짧은 요약 1줄로 통일. 본문 룰 / 예외 분기 / 코드 블록은 Checklist 에 미포함 (Process 본문 그대로 유지). Checklist 는 진행 상황 노출 전용.

### R2. og-* mirror 룰 약화 회귀

D3 가 mirror 룰 일부 양보. 향후 다른 영역도 mirror 룰 약화 회귀 가능.

**해결**: CLAUDE.md 의 og-* mirror 룰 본문에 명시 "**Checklist 섹션 한정 추가만** 예외. 다른 영역 (Procedure / Anti-Patterns / Related Skills / 식별자 / 영어 본문) 변경 절대 X". upstream 본문 변경 시점 검출 grep 도 그대로 유지.
