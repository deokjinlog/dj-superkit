## §1. Architecture overview

### 변경 패턴 (9 skill 공통)

각 skill body 의 `## Process` 섹션 직전 또는 직후에 `## Checklist` 섹션 신규 추가. 본문 다른 영역 (Anti-Patterns / Related Skills / 룰 본문) 변경 절대 X.

```markdown
## Checklist

- [ ] Step 1 — <Step 1 헤더 + 짧은 요약>
- [ ] Step 2 — <Step 2 헤더 + 짧은 요약>
- ...
```

### 흐름

1. 사용자가 슬래시 명령 또는 자연어로 skill 호출
2. using-superpowers 룰: "Has checklist? yes → Create TodoWrite todo per item"
3. 메인이 자동으로 TaskCreate N 번 호출 (N = Checklist 항목 수)
4. 각 Step 진행 시 TaskUpdate 로 in_progress / completed 전환
5. 사용자 시야에 진행 상황 자동 노출

### 신규 룰 도입 X

- 새로운 분기 / 새로운 게이트 / 새로운 도구 호출 패턴 없음
- 기존 Process 흐름을 사용자 시야에 노출하는 단순 패턴
- 메인 행동 변화: TaskCreate 도구 호출이 자동 발동 (현재는 메인이 수동 결정)

## §5. Key decisions

### D-1. 대상 9 skill 확정

requirements.md D1 답습. 사용자가 14 후보 중 명시 9 선택.

### D-2. Checklist 위치 = `## Process` 직전

**Alternatives**:

- A. Process 직전 (헤더 양식 일관) — Recommended
- B. Process 직후 (Step 본문 읽고 Checklist 확인)
- C. 본문 끝 (Anti-Patterns / Related Skills 다음)

**Recommendation**: A. 메인이 skill body Read 시 Checklist 를 먼저 만나 TaskCreate 자동 발동. 그 다음 Process 본문 따라 실행. 시각적 일관성 ↑.

### D-3. Checklist 항목 형식 = Step 헤더 + 짧은 요약 1줄

**Alternatives**:

- A. Step 헤더만 (예: `- [ ] Step 1`) — 너무 짧음
- B. Step 헤더 + 짧은 요약 1줄 (예: `- [ ] Step 1 — Slug 추론 + 폴더 생성`) — Recommended
- C. Step 헤더 + 본문 일부 (여러 줄) — 너무 김

**Recommendation**: B. 사용자 시야에 의미 있는 단위 + 메인 turn 의 task 화면에서 가독성 ↑.

### D-4. og-* 2 mirror 룰 명시 예외

requirements.md D3 답습. CLAUDE.md 의 og-* mirror 룰 본문에 "**Checklist 섹션 한정 추가만** 예외" 명시 추가. 다른 영역 변경은 여전히 차단.

### D-5. dogfood 시점 — 본 세션 즉시 + 다음 세션 이어서

본 세션이 v2.5.2 release 진행 자체가 dogfood. 단 본 세션에선 9 skill body 가 아직 Checklist 없음 → 메인이 수동 TaskCreate (v2.5.1 답습). 다음 세션 진입 시 (push + reload 후) 자동 TaskCreate 발동 — 진짜 dogfood.

## §6. Preliminary risks

### R-1. 각 skill 의 Step 수 다양해서 UX 일관성 ↓

requirements.md R1 답습. 해결: Step 헤더 + 짧은 요약 1줄로 통일. 본문 룰 / 코드 블록은 Checklist 미포함.

### R-2. og-* mirror 룰 약화 회귀

requirements.md R2 답습. 해결: CLAUDE.md 의 mirror 룰 본문에 "Checklist 한정 예외" 명시 + upstream 본문 변경 검출 grep 유지.

### R-3. TaskCreate 폭주 우려 (9 skill 각각 5~10 task 자동 발동)

대상 skill 들이 chain 호출 (auto-* 4 chain) 되면 task 누적 ↑.

**완화**: 각 skill 종료 시 TaskUpdate(completed) 자동 룰 답습 (기존 룰 보존). chain 의 다음 skill 진입 시 이전 skill 의 task 가 모두 completed 상태라 active task 수는 항상 한 skill 분량만.

### R-4. Checklist 항목 본문이 실제 Process Step 헤더와 drift

수동 작성이라 향후 Process Step 본문 갱신 시 Checklist 본문 갱신 누락 위험.

**완화**: CLAUDE.md 결합 메모에 "Process Step 헤더 변경 시 Checklist 항목 본문 동기화 필수" 명시 + 회귀 catch grep (Step N 명칭 일치 확인).
