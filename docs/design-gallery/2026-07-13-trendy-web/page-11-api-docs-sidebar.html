Vectors 업서트

벡터를 컬렉션에 삽입하거나 갱신합니다. 동일한 id가 이미 존재하면 값과 메타데이터를 덮어쓰고, 없으면 새로 생성합니다. 한 요청에 최대 1000개의 벡터를 배치로 처리할 수 있습니다.

요청 예시

Authorization 헤더에 API 키를 담아 전송합니다. 아래 탭에서 언어별 예시를 확인하세요.

요청 
curl -X POST https://api.vecta.io/v1/vectors/upsert \
  -H "Authorization: Bearer $VECTA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "namespace": "products",
    "vectors": [
      { "id": "sku_8821", "values": [0.021, -0.44, 0.87], "metadata": { "brand": "orbit" } }
    ]
  }'
import { Vecta } from "@vecta/sdk";

const vecta = new Vecta(process.env.VECTA_API_KEY);

await vecta.vectors.upsert({
  namespace: "products",
  vectors: [
    { id: "sku_8821", values: [0.021, -0.44, 0.87], metadata: { brand: "orbit" } }
  ]
});
from vecta import Vecta

client = Vecta(api_key=os.environ["VECTA_API_KEY"])

client.vectors.upsert(
    namespace="products",
    vectors=[
        {"id": "sku_8821", "values": [0.021, -0.44, 0.87], "metadata": {"brand": "orbit"}},
    ],
)

응답

성공 시 업서트된 벡터 수와 처리에 걸린 시간을 반환합니다.

본문 파라미터

요청 본문은 JSON 객체입니다. 중첩 필드는 vectors[].field 형식으로 표기합니다.

파라미터 타입 필수 설명
namespace string 선택 벡터를 저장할 네임스페이스. 생략하면 default에 기록됩니다.
vectors array 필수 업서트할 벡터 객체의 배열. 한 요청에 최대 1000개까지 허용합니다.
vectors[].id string 필수 벡터의 고유 식별자. 같은 값이 있으면 기존 벡터를 덮어씁니다.
vectors[].values float[] 필수 임베딩 값 배열. 차원 수는 컬렉션 설정과 정확히 일치해야 합니다.
vectors[].metadata object 선택 검색 시 필터링에 쓰이는 키-값 쌍. 값은 문자열·숫자·불리언만 지원합니다.
차원 불일치는 422로 거부됩니다. 컬렉션 생성 시 지정한 차원과 다른 길이의 values를 보내면 배치 전체가 실패합니다. 업서트 전 임베딩 모델의 출력 차원을 확인하세요.
