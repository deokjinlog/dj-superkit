# v2.0.3 — 8 dj-superkit skill 의 AskUserQuestion 도구 호출 강제

> **Design doc** (og-brainstorming output, free-form Socratic). Terminal handoff: `og-writing-plans` → `og-executing-plans` 또는 upstream `subagent-driven-development`.

## 한 줄 요약

dj-superkit 신규 4 skill (`brainstorming` / `designing-direction` / `writing-plans` / `executing-plans`) + auto-* 변형 4 skill 의 **모든 사용자 질문** 을 `AskUserQuestion` 도구로 강제. 산문 한 줄 질문 금지. 알람 (`Notification.elicitation_dialog` 매처 → `repeat-alert.sh` 30s 반복) 발화 보장으로 사용자 마찰 회복.

## 배경

`docs/backlog/v2.0.3-ask-user-question-mandatory.md` — 사용자 신고:

> "그 yes/no 받을때 뭔가.. 그 도구호출은 하지않고 그냥 'cycle3-tech-design.md is finalized. Proceed?' 이렇게 질문하니까 ... 사용자가 인지하기가 힘들어" (v1.1.8 신고 재발)

v1.1.8 에서 AskUserQuestion 도구 도입 + 주요 명시 게이트 (`Gate #8` / `Gate #11` / `Gate #14` 등) 적용. 그러나 **흐름 중간 즉흥 clarifying / Socratic 비-게이트 질문** 은 prose 로 남음. macOS Notification 훅 `elicitation_dialog` 매처가 prose 질문 catch 불가 → `repeat-alert.sh` 30s 반복 알람 발화 X → 사용자 다른 작업 중이면 질문 놓침.

본 세션 (v2.0.2 dogfood) 자체에서도 메인이 Q1, Q2, Q3 등 prose 질문 다수 던짐 → 룰 정당화 증명.

### 2. Socratic 자유응답 trigger — single Q AskUserQuestion + dummy choice

```python
AskUserQuestion(
    questions=[{
        "question": "<원하는 질문>",
        "header": "<짧은 라벨>",
        "options": [{"label": "알겠음", "description": "자유 응답 모드 — 다음 turn 에 prose 로 답해주세요"}],
        "multiSelect": False
    }]
)
```

- AskUserQuestion 호출 자체로 `elicitation_dialog` 매처 trigger → 알람 발화
- 사용자가 "알겠음" 선택 후 다음 turn 에 prose 자유 답변
- 또는 사용자가 "Other" 옵션 (도구 자동 제공) 으로 직접 답변

### 3. multi-question batching — 의미 결합 시 max 4 questions[]

허용 예시:
- 같은 결정 단계의 multi-axis (예: PRD 모드 선택 + Visual Companion 동의 동시)
- `Gate #11` 통합 승인 (doc 승인 + verify 보고서 확인 동시)

불허 예시:
- 다른 결정 차원 (예: 피처 이름 + 모드 선택 동시 X — 피처 이름 먼저 잡고 모드 선택)
- 의존성 있는 Q (한 답이 다른 Q 의미를 바꿈)

### 4. Boilerplate 위치 + 본문

8 skill body 각각의 **상단 section** ("# 제목" 후 / "<HARD-GATE>" 또는 "## Checklist" 전) 에 박음:

```markdown
## 사용자 질문 룰 (v2.0.3+) — 항상 AskUserQuestion

이 skill 흐름 안에서 사용자에게 질문할 일이 생기면 **반드시** `AskUserQuestion`
도구로 호출한다. 산문으로 "~ 할까요?" 한 줄 던지지 마라.

### Why

Notification 훅 (`elicitation_dialog` 매처) 이 알람을 발화하려면 도구 호출이
실제로 일어나야 함. 산문 질문은 훅이 못 잡아서 사용자가 놓침 (v1.1.8 신고 재발).

### How to apply

- clarifying / Socratic / 모호점 확인 / 게이트 / 모드 선택 — 모두 포함
- 단답 yes/no 도 prose X → `AskUserQuestion` choices `[yes, no]` 사용
- 다중 선택은 enum choices 또는 multi-question batching (의미 결합 시 max 4 questions[])
- **Socratic 자유 응답**: AskUserQuestion 의 question 본문에 "자유롭게 답해주세요" + dummy choice `[알겠음]` 1개 → 트리거만 발화, 응답은 다음 turn prose
- **예외**: 본문 자체의 알람-friendly 안내문 (`ℹ️ Auto-proceeding ...`) 는 질문 아니라 안내 — 도구 호출 불필요
```

## 우려 / Mitigation

### O-1. 알람 피로 — 호출 빈도 ↑

호출당 alarm trigger. 8 skill 흐름 한 turn 안 multi 호출 가능.

**Mitigation**: `repeat-alert.sh` 의 max=10 카운터 정책 (이미 설정됨). multi-Q batching 으로 한 호출에 묶기.

### O-2. 자연스러운 prose 흐름 깨짐

og 스타일 자유 응답이 도구 호출로 변환 시 어색함 가능.

**Mitigation**: Socratic 자유응답 dummy choice 패턴 — AskUserQuestion 발화는 알람만, 응답은 다음 turn prose. 강제 enum 선택 부담 없음.

### O-3. 8 skill 동기화 비용

boilerplate 변경 시 8 곳 동시 patch.

**Mitigation**: CLAUDE.md 결합 메모 + atomic 변경 룰 (v2.0.0/v2.0.1 패턴 재활용). grep 회귀 catch 룰.

### O-4. 일부 안내문 (`ℹ️ Auto-proceeding`) 과 충돌 X

안내문은 질문이 아니라 정보 노출. 도구 호출 불필요.

**Mitigation**: boilerplate 본문에 "예외: 안내문은 도구 호출 X" 명시.

### O-5. og-brainstorming 자체는 이 룰 적용 X

본 design doc 작성 중인 og-brainstorming 흐름 자체는 upstream-mirror — 룰 적용 X. v2.0.3 이 dj-superkit 신규 skill 만 대상.

**Mitigation**: design doc + implementation plan 단계에서 명시. 사용자가 og-* 흐름의 prose 질문 받아도 의도된 동작.
