# AI 디자인은 왜 맨날 똑같이 생겼을까

> 도메인 하나를 **12가지 스타일**로 뽑아 비교하는 클로드 스킬을 만들었다. 잘 된 이야기가 아니라, 만들면서 뭘 틀렸고 뭘 배웠는지의 기록이다.

AI한테 "대시보드 화면 만들어줘" 하면 매번 비슷하게 생긴 게 나온다. 파랑, 카드, 둥근 모서리, 그림자. 왜 그럴까 궁금했고, 그래서 **같은 데이터를 12가지 완전히 다른 스타일로 뽑아 나란히 비교하는 도구**를 만들어봤다.

---

## 결과물 먼저

콘텐츠는 똑같다. 스타일만 12개다. (e스포츠 팀 관제 대시보드 "ProGuild" 하나로 뽑았다)

(이미지: 12스타일 갤러리 한 화면 — gallery.html 캡처)

- **SF HUD** — 새로고침하면 부팅 시퀀스와 함께 전원이 들어온다. 어둠이 60%, 발광은 3곳.
- **브루탈리즘** — 초대형 숫자, 각진 원색, hover하면 쿵 찍힌다.
- **럭셔리 다크** — 골드 선이 그려지고, 세리프 대형 수치가 주인공. 모든 모션 500ms 이상.
- **뉴모피즘** — 클릭하면 표면이 실제로 눌린다.

(이미지: SF HUD vs 럭셔리 다크 — 같은 데이터, 정반대 무드 나란히)

같은 숫자인데 이렇게까지 갈라진다. 어떻게 여기까지 왔는지가 이 글의 내용이다.

---

## 왜 AI 디자인은 다 비슷한가

원인을 추정해봤다. 답은 **"매력 축"이 없어서**였다.

좋은 스타일에는 "왜 매력적인가" 하나의 축이 있다. SF HUD는 연극성, 공공포털은 신뢰, 럭셔리는 품격. 그냥 "예쁘게"라고 하면 AI는 안전한 평균값 — 파랑·카드·그림자 — 로 수렴한다. 어중간한 절충이 제일 안 매력적이다.

그래서 스타일마다 **매력 축 + 금지 목록**을 박았다. 금지가 핵심이다. 금지는 스타일 정체성의 절반이다.

```
[브루탈리즘 금지]
그라디언트, 블러, 부드러운 그림자, 둥근 모서리(radius 0), 파스텔
```

트렌디의 통통 튀는 카운트업을 럭셔리에 넣으면? 품격이 이벤트 당첨 화면이 된다. 금지 목록이 이 "기법 이식"을 막는다.

---

## 삽질 1 — 스타일이 오염됐다

12개를 병렬로 뽑기로 했다. 서브에이전트 하나가 스타일 하나씩. 그런데 결과가 어정쩡했다. 브루탈리즘인데 어딘가 글래스모피즘 냄새가 났다.

소거 끝에 남은 원인 — 에이전트가 **여러 스타일 파일을 같이 읽고 있었다.** 참조가 섞이니 스타일이 평균으로 끌려갔다.

해결은 단순했다. **각 에이전트는 자기 스타일 파일 하나만 읽는다.** 다른 파일은 절대 안 본다. 오염원을 물리적으로 차단하니 무드가 선명해졌다.

---

## 삽질 2 — 린터가 거짓말을 했다

12개를 한 번에 뽑으면 사람이 하나하나 품질 검수를 못 한다. 그래서 크리틱을 기계화했다. 각 스타일의 금지 조항을 검사하는 `style-lint.py`를 붙였다. 생성 → 린트 → 위반 수정 → 재린트. 결정론적 게이트다.

그런데 이 린터가 멀쩡한 디자인을 자꾸 FAIL 시켰다.

**브루탈리즘이 "그라디언트 금지" 위반이란다.** 열어보니 CSS에 `gradient()` 함수가 없었다. 린터가 주석 안의 단어 "gradient"를 잡은 거였다.

```python
# Before — 단어를 잡음
lambda s: has(s, r"gradient")
# After — 실제 CSS 함수만
lambda s: has(s, r"(?:linear|radial|conic)-gradient\s*\(")
```

같은 방식으로 두 개가 더 나왔다. `monospace`를 폰트 선언 밖에서도 잡던 것, 그리고 **`animation:none`을 애니메이션으로 오인하던 것.** `animation:none`은 오히려 애니메이션을 *끄는* reduced-motion 스위치인데, 린터는 "animation"이란 글자만 보고 위반이라 했다.

정량으로: 실전 테스트 한 번에 린터 버그 **3개**를 잡았다.

---

## 검증 — 진짜 도메인으로 돌려봤다

데모용 말고 진짜 도메인(e스포츠 관제)으로 12개를 병렬로 다시 뽑았다. 서브에이전트 13개(12 스타일 + 갤러리)가 동시에 돌았다.

결과:
- **콘텐츠 고정 12/12** — 12개 다 같은 데이터(승률 68.4%, 누적 상금 1억 8,400만원…). 스타일만 다르다.
- **린트 10/12 통과**, 2개 FAIL.

이 2개가 재밌었다. 하나(공공포털)는 위의 `animation:none` 오탐 — 린터 잘못. 고쳤다.

다른 하나(SF HUD)는 **린터가 맞았다.** 다크 배경이 패널에만 있고 `body`는 흰색이었다. HUD는 배경이 어두워야 하는데, 흰 여백이 샐 수 있었다. 린터가 정확히 잡은 거다.

```
[FAIL] style-01-sf-hud.html  ✗ 흰/밝은 배경 금지 (HUD는 어둠)
```

body에 다크 배경 한 줄 추가하고 재린트했다.

```
총 위반: 0건 → 통과 ✅   (12/12 PASS)
```

(이미지: 린트 FAIL → 수정 → 12/12 PASS 콘솔)

생성 → 린트 → 수정 → 통과. 이 루프가 실제로 돈다는 걸 이때 확인했다. "느낌상 괜찮아 보임"과 "검증된 통과"의 차이가 여기 있었다.

---

## 정리

- AI 디자인이 평범한 건 **매력 축과 금지가 없어서**다. 있으면 12개가 확 갈린다.
- 병렬로 뽑을 땐 **각자 자기 것만 읽게** 격리해야 스타일이 안 섞인다.
- 크리틱을 사람 눈에 맡기면 편차가 크다. **린터로 기계화**하면 12개가 다 검증된 채로 나온다 — 단, 린터도 틀리니 실전으로 검증해야 한다.

구조는 이렇게 정리됐다.

```
skills/design-style-explorer/
├── SKILL.md
├── references/
│   ├── concepts.md          # 매력 축·금지 철학
│   ├── master-prompt.md
│   └── prompts/style-01~12.md
└── scripts/style-lint.py    # 금지 자동 검사 게이트
```
